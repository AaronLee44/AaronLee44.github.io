Generate Parentheses

题目描述

Given n pairs of parentheses, write a function to generate all combinations of well-formed parentheses.
For example, given n = 3, a solution set is:
$”((()))”, “(()())”, “(())()”, “()(())”, “()()()”$
题目大意：给定$n$对括号，将它们所有可能的组合方式打印出来。

思路

在一次正确的打印括号的过程中，我们知道已经打印出来的括号中，左括号的数目一定大于等于右括号的数目，否则括号将不会匹配。所以就可以得知剩余需要打印的括号中，左括号的数目一定小于等于右括号的数目。假设给定$n=k$,则总共需要打印$2k$个括号（$k$个左括号和$k$个右括号），我们用$i$表示打印第$i$个括号这一操作，对于这一状态，我们知道最多有两种选择：打印左括号（如果左括号没有打印完的话）或者打印右括号。只要左括号没有打印完，我们就可以选择打印左括号。但是什么时候可以打印右括号呢？可以发现只要右括号没有打印完并且剩余右括号的数目大于左括号的数目就可以打印。总而言之，只要打印出的括号不会违背括号匹配原则即可。
从上面的分析过程可以发现，从第$i$个状态到第$i+1$个状态是一个递归的过程，而递归过程所需要遵守的条件就是上述括号打印所遵守的条件。
如下图所示，整个过程是一个二叉树，从根节开始一直往下，红色部分即是不满足括号匹配原则的无效路径。

取模操作（Modulo operation)

概述

“取模(Modulo)”操作总是容易让人觉得定义模糊，当两个操作数符号相同时，不存在问题；但是当两个数符号相异时就会出现差别。使用C++的人会发现python中取模运算不同于C++，同理其他语言也会遇到同样的情况。而这一问题的主要原因在于不同编程语言采用的定义大不相同，有的语言甚至在标准里面没有明确的定义，而是将其交给编译器人员去实现。
维基百科上对取模操作的原始定义为：

Given two positive numbers, a (the dividend) and n (the divisor), a modulo n (abbreviated as a mod n) is the remainder of the Euclidean division of a by n.

原始的定义中两个操作数均为正数，而在计算机领域则会涉及两个数异号的情况，这时候取模操作的实现就取决于编程语言或是其他因素（比如硬件）。
维基上的原话为：

In mathematics the result of the modulo operation is the remainder of the Euclidean division. However, other conventions are possible.
Computers and calculators have various ways of storing and representing numbers; thus their definition of the modulo operation depends on the programming language and/or the underlying hardware.

两种较多使用的定义

定义1：$r=a-n*\lfloor\frac{a}{n}\rfloor$，其中$r$表示余数，$a$为被除数，$n$为除数，符号$\lfloor{x}\rfloor$为表示向下取整(floor fuction).

例子：计算$a/n$的时候是向下取整($\lfloor\frac{a}{n}\rfloor$),即取不大于运算结果的最大整数，例如：-4/3=-1.333…对该结果向下取整得到-2（即不大于-1.333…的最大整数），此时$\lfloor\frac{a}{n}\rfloor=-2$，然后 $r=-4-3*(-2)=2$.

定义2：$r=a-n*\lceil\frac{a}{n}\rceil$，其中$r$表示余数，$a$为被除数，$n$为除数，符号$\lceil{x}\rceil$为表示向上取整(ceiling fuction).

例子：计算$a/n$的时候是向上取整($\lceil\frac{a}{n}\rceil$)，即取大于运算结果的最小整数，例如：-4/3=-1.333…对该结果向上取整得到-1（即大于-1.333…的最小整数），所以$\lceil\frac{a}{n}\rceil=-1$，从而 $r=-4-3*(-1)=-1$.

C++和Python中的取模

C (ISO 1990) div Dividend
C (ISO 1999) %, div Dividend
C++ (ISO 1998) % Implementation-defined
C++ (ISO 1998) div Dividend

  • 可以看到早期的C和C++，模运算(%)的结果符号是Implementation-defined的，即标准并不规定如何实现，一般将这种实现交给编译器人员。到后期以后就规定了具体的实现方式，即结果符号都和被除数相同。C/C++中的取模操作除了$\%$操作符之外还有div函数。
  • 对于Python，$\%$的结果和除数相同。另外Python还定义了math.fmod方法，其结果的符号与被除数相同。

Container With Most Water

问题描述

Given n non-negative integers a1, a2, …, an, where each represents a point at coordinate (i, ai). n vertical lines are drawn such that the two endpoints of line i is at (i, ai) and (i, 0). Find two lines, which together with x-axis forms a container, such that the container contains the most water.
问题大意：如下图所示，横坐标的每个$i$对应一个$a_i$，从中选取两个，使它们之间存储的水的容量最大。

分析

我们可以知道，最大容量一定由某两个垂线围成（废话！ $(＞﹏＜)$ ），现在的问题是如何寻找到这两条垂线。如果我们使用两个指针，使第一个指针开始时指向头部$a_1$，第二个开始时指向$a_n$，然后让第一个指针从前往后扫描，第二个指针从后往前扫描。
现在我们维护$a_i$和$a_j$围成的区域，我们认为它们之间存储的容量最大，而它们所存储的容量是由其中线段最短的一方决定，所以如果某一个线段的长度小于另外一个，那么我们应该去替换掉短的那一方，即让短的一方指针向后步进（对于指针i）或者向前步进（对于指针j）。然后比较指针改变前和改变后容量大小的变化，如果容量变大，则存储这一容量值。
这样，当两个指针相遇的时候我们就可以知道，最大值一定存储在之前维护的那个“最大值的变量”之中。

反转一个整数的各个位上的数字

题目说明

这是一道leetcode上的一道算法题，输入一个32位整数，要求反转整数的各个数位，例如输入123，则输出321；输入-123，则输出-321。

题目分析

这道题目主要问题在于32位int的溢出，32位int可以表示的范围为$-2^{31}$~$2^{31}-1$，即-2147483648～2147483647。如果输入的整数在反转之后会导致溢出，则返回0。 溢出问题可以通过将反转之后的数存入long long类型中来避免复杂的判断。

解决思路

方法1：

可以迭代的对输入的数字与10进行取模(%)和除法运算(/)，取模可以得到得到该数个位上的数字，除以10可以将之前的个位去掉，使得十位成为新的个位，然后不断迭代，在该过程中只要注意溢出问题即可。

方法2：

利用C++的string流可以专注于对每个数位的操作，而不用在意类型的转换问题。我们可以将要反转的数字存入一个string流对象，然后将string流中的数字输出到一个string变量，对该string变量从末尾输出到另一个string对象就可以完成字符串的反转，最后只需要用string流将反转后的数字转换为int类型并考虑溢出问题即可。

整数排序

问题描述

实现输入一组大于等于0的整数，根据从小到大的顺序排序后输出，排序后有连续数时，只输出连续数中最小和最大的两个数。
输入: 一组大于等于0的整数，不考虑非法输入，各个整数之间以逗号(“,”)分隔, 输入字符串的总长度小于等于100个字节。
输出: 排序后的值，各个整数之间以空格分隔。
样例输入: 1,4,3,110,2,90,7
样例输出: 1 4 7 90 110

题目分析

问题可以分为两步来实现：首先进行字符串处理，去掉字符串中的逗号，提取出数字。然后从小到大进行排序，排序之后对连续数进行分析，只保存其中最大和最小的两个数，然后在数字之间添加空格后输出。

字符串处理部分通过寻找逗号的位置，提取出逗号之间的数字。
问题难的地方在于如何处理连续数字，只保存其中最小值和最大值。在对提取出的数字进行排序之后，数字按从小到大排列，由于连续数字之间的差值为1，我就可以通过计算前一个数字和后一个数字之间的差值来判断连续的数字串，然后使用一个continueCount对连续的数字进行计数，当continueCount=2时，我们就知道了出现了需要删除的中间数字，然后我们从容器中删除它们并对continueCount值减1.

大数阶乘

计算大数阶乘的思路

由于计算阶乘的过程中，主要问题是——数字不断相乘之后由于结果剧增使普通的C++整型数据类型无法存储，所以我们可以定义一个int类型的数组来存储计算的结果，数组的每个单元都只保存一位，例如arr[0]只保存个位，以此类推。这样，我们在计算$12…*n$过程中，先将arr[0]设置为1，其他元素均为0，然后用2乘以整个数组，考虑每一位上的进位和余数，然后更新这个arr[i]单元，最后迭代即可。

如下所示，开始时设置最低位arr[0]=1，下面为$n=1$时的结果：

arr[N-1] … … … … arr[2] arr[1] arr[0]
0 0 0 0 0 0 0 1

当$n=2$时，用$2$乘以上面整个数组，得到：

arr[N-1] … … … … arr[2] arr[1] arr[0]
0 0 0 0 0 0 0 2

当$n=3$时，用$3$乘以上面整个数组，得到：

arr[N-1] … … … … arr[2] arr[1] arr[0]
0 0 0 0 0 0 0 6

当$n=4$时，用$4$乘以上面整个数组，数组更新为：

arr[N-1] … … … … arr[2] arr[1] arr[0]
0 0 0 0 0 0 2 4

…
以此类推，在此过程中值需要考虑每一位上的进位和进位之后的余数。最后数组中每个单元的数字就是阶乘结果的每一位。

N个台阶问题

题目描述

有一个具有N个台阶的阶梯，一次可以上一阶或是两阶，问有多少中走法。

题目分析

方法1：

这道题最简单的解决思路是采用递归的思想，从最后一步开始分析，先不去考虑先前的走法，对于最后一步可能的走法只有两种：只需走一阶到达第N个台阶或需要走两阶到达第N个台阶。然后分别考虑最后一步走一阶的情况和走两阶的情况，问题变为一个递归问题。
对于走一阶的情况：变为求前面N-1个台阶如何走；同样对于走两阶的情况：问题变为求前面N-2个台阶如何走的问题。
使用数学描述：假设对于N阶的台阶有f(N)种走法，如果最后一步走一阶，则剩余的N-1阶的走法数目为f(N-1);类似的，如果最后一步走两阶，则剩余N-2个台阶走法的数目为f(N-2)。所以N个台阶总共的走法：f(N)=f(N-1)+f(N-2)。由此得到了一个递归定义的公式，现在只需要考虑递归结束条件即可解决问题。递归计算到f(2)+f(1)时结束，由题意可以得知f(1)=1,f(2)=2。

问题：上述代码采用了递归的方法求解，但是由于递归本身时间和空间消耗都比较大，所以程序并不是很高效。
观察公式f(N)=f(N-1)+f(N-2)可以发现，要计算N个台阶的走法只需要知道N-1和N-2个台阶的走法，所以实际上只需要存储这两个状态的信息即可，因此我们可以尝试将递归程序改写为迭代的方式。如下所示，时间复杂度变为了O(N)。

方法2：

该问题还可以采用数学的方法直接解决，假设一次走一阶的共有$x$次，走两阶的共有$y$次，则可以得到$x+2y=N$，我们可以对$x$从0到N进行遍历，将每一个$x$代入前面的等式求出$y=\frac{N-x}{2}$，如果$y$是整数，则保存这一对$(x,y)$。然后考虑对于每一对$(x,y)$它们的排列问题即可求出这一对$(x,y)$的所有走法，然后将所有满足条件的$(x,y)$的走法相加就可以得到总的走法数目。
对于某一对$(x,y)$，求其所有排列的数目是一个捆绑插空问题，即一次走两步（$y$）的情况要看做整体（捆绑在一起），所以总共有$x+y=N-2y+y=N-y$个空，即需要从$N-y$个空中选出$x$个插入就可以得到这一对$(x,y)$的所有走法的组合$C_{N-y}^y$。
